EXPLICACIÓN DE LA CARPETA .MVN

La carpeta .mvn es parte del Maven Wrapper, una herramienta que permite ejecutar Maven sin necesidad de tenerlo instalado globalmente en el sistema. Esto asegura que todos los desarrolladores usen la misma versión de Maven, evitando problemas de compatibilidad. La carpeta contiene scripts y archivos para descargar y ejecutar Maven de manera independiente. A continuación, se detalla el contenido de esta carpeta basado en el análisis del proyecto SuperGO2-Server.

1. ESTRUCTURA DE LA CARPETA .MVN:
   - wrapper/: Subcarpeta que contiene los archivos principales del wrapper.
     - maven-wrapper.jar: Archivo JAR que actúa como el ejecutable principal del wrapper. Este JAR se descarga automáticamente si no existe y se usa para ejecutar comandos de Maven.
     - maven-wrapper.properties: Archivo de propiedades que configura el wrapper. Incluye:
       - distributionUrl: URL para descargar la distribución de Maven (en este caso, Apache Maven 3.6.3 desde el repositorio oficial).
       - wrapperUrl: URL para descargar el maven-wrapper.jar si es necesario.
     - MavenWrapperDownloader.java: Código fuente Java que se usa para descargar el maven-wrapper.jar si no está presente. Este archivo es el descargador que se ejecuta para obtener el JAR necesario.

2. ARCHIVOS PRINCIPALES:
   - maven-wrapper.properties: Este archivo de texto plano contiene configuraciones clave:
     - distributionUrl=https://repo.maven.apache.org/maven2/org/apache/maven/apache-maven/3.6.3/apache-maven-3.6.3-bin.zip: Especifica la versión y URL de la distribución de Maven a usar (Maven 3.6.3).
     - wrapperUrl=https://repo.maven.apache.org/maven2/io/takari/maven-wrapper/0.5.6/maven-wrapper-0.5.6.jar: URL para descargar el wrapper JAR si no existe localmente.
     Este archivo permite personalizar la versión de Maven sin modificar scripts.

   - MavenWrapperDownloader.java: Este es el código fuente de la clase que maneja la descarga del wrapper. Incluye:
     - Constantes para la versión del wrapper (0.5.6), URLs por defecto y rutas de archivos.
     - Método main que lee el directorio base, carga propiedades de maven-wrapper.properties, y descarga el JAR desde la URL especificada.
     - Manejo de autenticación si se proporcionan variables de entorno (MVNW_USERNAME y MVNW_PASSWORD).
     - Descarga del archivo usando canales NIO para eficiencia.
     - Logging básico para rastrear el progreso (aunque usa BotLogger, que parece ser una clase personalizada).
     Este archivo se compila y ejecuta automáticamente cuando se necesita el wrapper.

   - maven-wrapper.jar: El JAR compilado del descargador. Se genera a partir de MavenWrapperDownloader.java y se usa para ejecutar Maven sin instalación global.

3. PROPÓSITO GENERAL:
   - El Maven Wrapper permite que el proyecto sea "autónomo" en términos de Maven, asegurando que comandos como mvn compile o mvn package usen la versión correcta de Maven.
   - Scripts como mvnw (en la raíz del proyecto) usan esta carpeta para invocar Maven.
   - Es especialmente útil en entornos donde Maven no está instalado o donde se necesita una versión específica para evitar conflictos.
   - En proyectos colaborativos, .mvn se incluye en el repositorio para que todos usen la misma configuración.

4. USO EN EL PROYECTO:
   - En SuperGO2-Server, el wrapper asegura que se use Maven 3.6.3, compatible con las dependencias y plugins definidos en pom.xml.
   - Facilita el desarrollo en diferentes sistemas operativos (Windows, Linux, macOS) sin requerir instalación manual de Maven.

Esta carpeta es fundamental para la portabilidad y consistencia del build del proyecto.
