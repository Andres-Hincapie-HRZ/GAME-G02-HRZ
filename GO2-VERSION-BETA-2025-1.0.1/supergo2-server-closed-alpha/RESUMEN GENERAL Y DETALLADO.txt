RESUMEN GENERAL Y DETALLADO DEL PROYECTO SUPERGO2-SERVER

1. DESCRIPCIÓN GENERAL:
   - Este es un proyecto de servidor backend para el juego "SuperGO2", un juego de estrategia en el espacio, desarrollado con Spring Boot (versión 2.6.3).
   - El artefacto principal es "game-server" con grupo "com.go2super" y versión "0.0.1-SNAPSHOT".
   - Utiliza Java 17 y herramientas como Lombok para reducir código boilerplate y MapStruct para mapeo de objetos.
   - El README.md es minimalista, solo contiene el título "# go2super-server".
   - El proyecto incluye soporte para perfiles de desarrollo (dev) y producción (pro), con configuraciones específicas para cada uno.

2. ESTRUCTURA DE CARPETAS PRINCIPALES:
   - .idea/: Configuración del IDE (IntelliJ IDEA), incluyendo archivos como compiler.xml, encodings.xml, jarRepositories.xml, misc.xml, etc., para facilitar el desarrollo.
   - .mvn/: Wrapper de Maven (mvnw y mvnw.cmd) para gestionar dependencias y builds sin necesidad de Maven instalado globalmente.
   - logs/: Contiene archivos de logs de la aplicación, como application-debug.log, para monitoreo y depuración en tiempo real.
   - markdowns/: Documentación en formato Markdown, incluyendo login-steps.md que describe los pasos de login (envío de paquete 502 y respuesta 504).
   - META-INF/: Metadatos de la aplicación, como configuraciones de Spring AOT para optimización nativa.
   - src/: Código fuente principal, organizado en paquetes Java bajo "com.go2super".
   - target/: Directorio de salida del build, incluyendo librerías compiladas (en target/lib/) y archivos generados por Maven.

3. ESTRUCTURA DEL CÓDIGO FUENTE (src/):
   - El código está organizado bajo el paquete "com.go2super".
   - Principales subpaquetes:
     - buffer/: Manejo de buffers para comunicación eficiente (e.g., Go2Buffer.java).
     - database/: Capa de base de datos con entidades (Account, User, Fleet, Commander, etc.), repositorios (usando MongoDB), cachés (AccountCache, FleetCache, etc.) y tipos enumerados (AccountStatus, PlanetType, etc.).
     - packet/: Clases para paquetes de comunicación en tiempo real, divididos en categorías como fight (batallas), match (partidas), payment (pagos), science (ciencia), ship (naves), social (social), station (estación), task (tareas), upgrade (mejoras). Incluye requests y responses para funcionalidades del juego.
     - resources/: Archivos JSON de datos del juego (builds, chips, commanders, constellations, corpsLevel, etc.), serialización (deserializadores para JSON), localización (Localization.java) y assets (JSON de naves, props).
     - service/: Servicios de lógica de negocio, como BattleService, ChatService, CommanderService, CorpService, DashboardService, DiscordService, EmailService, GalaxyService, JobService, LoginService, MetricService, PacketService, RaidsService, RankService, ResourcesService, RiskService, SessionService, SocketService, TaskService, TeamModelService, TradeService, UserService. Incluye subpaquetes como battle (Pathfinder), command (Command), exception (BadGuidException), jobs (tareas programadas como UnitTask, UserTask), raids (PaymentService), validation (RegisterValidation).
     - socket/: Utilidades para manejo de sockets, incluyendo paquetes de datos (BasicCommand, IncomingBuffer, IncomingPacket), herramientas como Crypto (encriptación), DateUtil, IPLocation, MathUtil, RandomUtil, SocketUtil, TimeUtil, y DTOs como IPLookupDTO.
     - utility/: Clases de utilidad general, como GlueList, ObjectUtility, RomanNumber.
   - Recursos: Incluye application.yml para configuración, assets (JSON de naves, props), datos del juego (JSON para builds, commanders, etc.), plantillas de email (otp.html) y localización (lang/en-us.json).

4. CONFIGURACIÓN (application.yml):
   - Perfiles: dev (puerto 9090, MongoDB local) y pro (variables de entorno para MongoDB).
   - Configuraciones de juego: Versión de parche (v2022.2.pro-dev), whitelist, habilitación de registro y login, modo de mantenimiento, IP pública, timeouts de socket (70s), intervalos (100ms), modos de prueba y transmisión rápida, verbose level (5), integración con GELF para logs, hCaptcha para seguridad, SMTP para emails (usando Mailgun), credenciales de admin, límite de usuarios (3), fase del juego (1).
   - Discord: Configuración de IDs de canales (owner, guild, cmd-channel, audit-channels para incidentes, chat, trade, etc.) y token del bot.
   - Logging: Nivel INFO para root y Spring, DEBUG para co.com.aruma, patrón de consola detallado.
   - Otras: Compresión de servidor habilitada, timezone GMT-5, live reload en dev.

5. DEPENDENCIAS Y TECNOLOGÍAS:
   - Framework principal: Spring Boot con starters para web, websocket, data-mongodb, mail, validation, logging, json, tomcat.
   - Base de datos: MongoDB con drivers (mongo-java-driver, mongodb-driver-sync, etc.) y RelMongo para relaciones.
   - Comunicación: Netty SocketIO para sockets en tiempo real, WebSockets, STOMP, WebJars para frontend (bootstrap, jquery, sockjs-client).
   - Otras librerías: Gson para JSON, Lombok para anotaciones, Jasypt para encriptación, Reflections para introspección, Joor para proxies, Guava para utilidades, Jansi para colores en consola, Javatuples para tuplas, Apache Mina para networking, Unirest para HTTP, Sematext para logging en la nube, JDA para Discord, OkHttp para HTTP, Commons Net para redes, Trove4j para colecciones eficientes.
   - Build: Configurado para generar imágenes nativas con Spring Native y GraalVM, usando Paketo Buildpacks.

6. FUNCIONALIDADES PRINCIPALES:
   - Servidor de juego con soporte para batallas (fight), partidas (match), pagos (payment), ciencia (science), naves (ship), modelos de naves (shipmodel), social (amigos), estación (actualización de nombre), tareas (task), upgrades (mejoras de naves).
   - Manejo de sesiones de usuario, autenticación y validación (RegisterValidation, hCaptcha).
   - Integración con servicios externos: Discord (para auditoría y comandos), email (para OTP y notificaciones), GELF para logs remotos.
   - Sistema de paquetes para comunicación cliente-servidor en áreas como combates, ciencia, naves y sociales, con enrutamiento (PacketRouter) y procesamiento (PacketProcessor).
   - Jobs programados: Para construcción de naves, upgrades, rankings, tecnología, etc.
   - Caches para optimización: AccountCache, FleetCache, etc.
   - Soporte para raids, rankings, comercio, y más, con datos JSON para configuraciones del juego.

7. PUNTOS ADICIONALES:
   - El servidor principal (GameServer.java) maneja conexiones de sockets, acepta clientes, procesa paquetes y gestiona sesiones.
   - Incluye soporte para cross-domain policy y timeouts de inactividad.
   - El proyecto enfatiza la comunicación en tiempo real, gestión de datos complejos y escalabilidad con MongoDB y cachés.

Este proyecto parece ser un servidor robusto y completo para un juego de estrategia en el espacio, con énfasis en la comunicación en tiempo real, integración con servicios externos y gestión eficiente de datos.
